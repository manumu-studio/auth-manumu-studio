## Resolution

### Contributing Factors

1. **Factor:** Packet 02 expanded the production-required env contract.
   - **How it contributed:** Preview must now carry Turnstile, worker-auth, invite-delivery, and Admin-MFA key variables.
   - **Why it was not caught earlier:** Local builds used shell-provided placeholders, while Vercel Preview had not yet been provisioned.
2. **Factor:** Vercel env readiness is external to the repository.
   - **How it contributed:** CI/local gates pass but cannot prove Vercel project settings are complete.
   - **Why it was not caught earlier:** There is no automated preflight comparing `src/lib/env.ts` production requirements against Vercel scopes.

### Was the Hypothesis Correct?

Yes. The exact Vercel build log lists the eight missing Packet 02 variables and `vercel env list preview` confirms they are absent from global Preview scope.

### Files Changed

| File | Change | Why |
|------|--------|-----|
| Vercel Preview environment settings | Pending manual addition of eight Packet 02 vars | Required to satisfy fail-closed build env validation |

### Fix Approach

Provision the missing Packet 02 env variables in Vercel Preview, then redeploy commit `6d34ec7`.
